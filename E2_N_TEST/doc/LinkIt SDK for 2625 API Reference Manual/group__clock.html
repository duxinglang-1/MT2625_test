HAL_CLOCK_CG_ID descriptions

Each hal_clock_cg_id is related to one hardware clock gate. Please check the following impacts before controlling the clock.

The description of API parameters for HAL_CLOCK_CG_ID is listed below:

HAL_CLOCK_CG_ID Details
HAL_CLOCK_CG_CM4_OSTIMER The clock for OS timer. It cannot be disabled, otherwise the system will fail.
HAL_CLOCK_CG_UART0 The clock for UART0. It cannot be disabled, otherwise the system will fail.
HAL_CLOCK_CG_SPISLV The clock for SPI slave. This clock should be enabled when it is connected to the master device if choosing a custom driver.
HAL_CLOCK_CG_GPCOUNTER The clock for general purpose counter timer. It cannot be disabled, otherwise the system will fail.
HAL_CLOCK_CG_CM_SYSROM The clock for system ROM. It cannot be disabled, otherwise the system will fail.
HAL_CLOCK_CG_SFC The clock for serial flash controller. It cannot be disabled, otherwise the system will fail.
HAL_CLOCK_CG_DISP_PWM The clock for display. This clock should be enabled if choosing a custom display driver.
HAL_CLOCK_CG_AAL The clock for Ambient-Light Adaptive Luma. This clock should be enabled if choosing a custom display driver.
HAL_CLOCK_CG_DSI0 The clock for display serial interface. This clock should be enabled if choosing a custom display driver.
HAL_CLOCK_CG_SENSOR_DMA The clock for sensor DMA. This clock should be enabled when using I2C0 or I2C1 with custom driver.
